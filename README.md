# 关键词采集 Edge 插件合集

这个仓库里目前放了两个可以直接加载到 Edge 的本地扩展：

- `edge插件/heybox`：小黑盒关键词采集插件。
- `edge插件/xiaohongshu`：小红书关键词采集插件。

两个插件都用于在已打开页面里扫描关键词、标记命中内容、管理多条评论草稿，并把随机草稿准备到对应帖子或笔记里。评论发布仍然需要你自己检查内容，并点击网站原生发布按钮。

## 把插件放入 Edge

### 1. 确认本地插件目录

当前可加载的插件目录是：

- 小黑盒：`E:\edge插件\heybox`
- 小红书：`E:\edge插件\xiaohongshu`

加载时一定要选上面的子目录，不要选 `E:\heybox`，也不要只选 `E:\edge插件`。如果选错目录，Edge 会提示清单文件缺失，或者插件加载后不能正常运行。

### 2. 打开 Edge 扩展管理页

1. 打开你平时使用的 Edge。
2. 在地址栏输入 `edge://extensions/`，按回车。
3. 打开页面左侧或左下角的「开发人员模式」开关。

注意：插件是按 Edge 配置文件安装的。如果你在测试 Edge、临时 Edge 或另一个用户配置里加载，常用 Edge 里不会出现；要在你平时用的那个 Edge 窗口里操作。

### 3. 加载插件

1. 在 `edge://extensions/` 页面点击「加载解压缩的扩展」。
2. 弹出文件夹选择窗口后，选择其中一个插件目录：
   - 要装小黑盒，选 `E:\edge插件\heybox`
   - 要装小红书，选 `E:\edge插件\xiaohongshu`
3. 点「选择文件夹」。
4. 如果两个插件都要装，再点一次「加载解压缩的扩展」，选择另一个目录。

加载成功后，扩展页会出现对应插件卡片：

- `Xiaoheihe Keyword Helper`
- `Xiaohongshu Keyword Helper`

### 4. 打开网站使用

- 小黑盒插件：打开 `https://www.xiaoheihe.cn/app/bbs/home`
- 小红书插件：打开 `https://www.xiaohongshu.com/explore`

进入对应网站后，页面右侧会出现插件面板。这个插件主要是页面内面板，不一定会像普通扩展那样只靠浏览器右上角图标操作。

### 5. 更新插件代码后怎么处理

GitHub 代码更新后，本地文件变了，但 Edge 不会自动刷新插件。需要手动刷新：

1. 打开 `edge://extensions/`
2. 找到对应插件卡片。
3. 点击卡片上的「重新加载」按钮。
4. 回到网站页面，刷新网页。

如果插件卡片不见了，或者你移动过文件夹位置，直接在扩展页删除旧插件，然后重新按上面的步骤加载对应目录。

### 6. 小红书权限提示

小红书插件为了实现真实自动填写，需要 Edge 的 `debugger` 权限。重新加载或重新安装时，如果 Edge 提示确认权限，选择允许。

这个权限只用于自动点击评论框、写入草稿文字；插件不会自动点击发布、发送、关注、点赞、收藏等按钮。填写完成后插件会主动断开调试连接。

## 小黑盒插件

- 打开 `https://www.xiaoheihe.cn/app/bbs/home`。
- 右侧面板输入关键词，每行一个。
- 可以添加多条评论草稿；每条草稿可以写多行。
- 「扫当前」只扫已经加载出来的帖子。
- 「采集更多」会向下滚动，边加载边扫描。
- 「打开搜索页」适合关键词比较偏、首页推荐流刷不到的情况。
- 「复制草稿」会随机复制一条草稿。
- 「打开并预填」会打开帖子并尝试把随机草稿填进评论框，最后仍需手动发布。

## 小红书插件

- 打开 `https://www.xiaohongshu.com/explore`。
- 右侧面板输入关键词，每行一个。
- 可以添加多条评论草稿；每条草稿可以写多行。
- 「扫当前」只扫已经加载出来的笔记。
- 「采集更多」会向下滚动，边加载边扫描。
- 「打开搜索页」适合首页推荐流刷不到目标内容的情况。
- 「复制草稿」会随机复制一条草稿。
- 「打开并预填」会打开笔记，自动展开评论框，并把随机草稿写入评论框。不会自动点击发布。

小红书插件为了实现真实自动填写，新增了 Edge 的 `debugger` 权限。重新加载插件时 Edge 可能会提示确认权限；填写完成后插件会主动断开调试连接。

## 扫不到时怎么调

- 先把关键词放宽，比如从完整短句改成 `求推荐`、`怎么买`、`多少钱` 这种词。
- 增大「采集页数」，再点「采集更多」。
- 关键词比较偏时，不要只扫首页推荐流，直接走站内搜索结果。
- 如果页面出现验证码、登录拦截或内容加载失败，先手动处理页面状态，再继续采集。
