# Xiaoheihe Keyword Helper

这是一个合规版浏览器扩展原型：自动扫描小黑盒页面里的关键词帖子，高亮匹配内容，并生成评论草稿供你手动复制、确认、发布。

它不会自动批量发评论，也不会绕过网站的交互流程。

## 安装

1. 打开你平时使用的 Edge。
2. 在地址栏输入 `edge://extensions/`，按回车。
3. 打开「开发人员模式」。
4. 点击「加载解压缩的扩展」。
5. 在文件夹选择窗口里选择：`E:\edge插件\heybox`
6. 点击「选择文件夹」。
7. 扩展页出现 `Xiaoheihe Keyword Helper`，就说明加载成功。
8. 打开 `https://www.xiaoheihe.cn/app/bbs/home`，页面右侧会出现插件面板。

不要选择 `E:\edge插件` 总目录，也不要选择仓库根目录 `E:\heybox`。Edge 需要加载包含 `manifest.json` 的具体插件文件夹，也就是 `E:\edge插件\heybox`。

如果你在常用 Edge 里看不到插件，说明它可能被加载到了另一个 Edge 配置文件或测试窗口里。请回到你平时使用的 Edge，重新打开 `edge://extensions/`，再加载一次上面的目录。

更新代码后，需要回到 `edge://extensions/` 点一次插件卡片上的「重新加载」，然后刷新小黑盒页面。

## 使用

- 右侧面板里输入关键词，每行一个。
- 在「评论草稿池」里添加多个草稿；每个草稿框都可以写多行。
- 点击「扫当前」只扫描已经加载在页面里的帖子。
- 点击「采集更多」会自动向下滚动，边加载边扫描更多帖子。
- 如果主页/话题页刷不到目标内容，点击「打开搜索页」。插件会复制第一个关键词并打开小黑盒搜索页；在已登录状态下粘贴搜索，再用「扫当前」或「采集更多」扫描搜索结果。
- 页面中匹配到的帖子会出现绿色描边，右侧列表会汇总命中结果。
- 点击「复制草稿」后，会从草稿池随机复制一个完整草稿，到帖子里自行确认并发布。
- 点击「打开并预填」会随机抽一个完整草稿，打开对应帖子，并尝试填到评论框里；最后仍需要你手动检查并点击小黑盒原生发布按钮。

## 扫不到时怎么调

- 先把关键词放宽，比如从完整短句改成 `买`、`求推荐`、`合适` 这种词。
- 增大「采集页数」，再点「采集更多」。
- 关键词比较偏时，不要扫主页推荐流，直接用「打开搜索页」走站内搜索结果。
- 如果页面出现验证码或登录拦截，先手动处理页面状态，再继续采集。

## 调整默认文案

默认关键词和草稿在 `content.js` 顶部。多个默认草稿写进 `defaultDrafts` 数组；单个草稿里可以保留换行：

```js
const defaultKeywords = ["买", "求推荐", "合适", "多少钱"];
const defaultDrafts = [
  "看情况回一句，发之前先确认帖子内容是否真的相关。",
  "这个得看具体需求。\n先确认帖子内容，再决定怎么回。"
];
```
