# Xiaohongshu Keyword Helper

这是一个合规版浏览器扩展原型：自动扫描小红书页面里的关键词笔记，高亮匹配内容，并准备评论草稿供你手动复制、打开预填、确认发布。

它不会自动批量发评论，也不会绕过网站的交互流程。

## 安装

1. 打开 Edge 的扩展管理页。
2. 开启「开发人员模式」。
3. 选择「加载解压缩的扩展」。
4. 选择本目录：`xiaohongshu`。
5. 打开 `https://www.xiaohongshu.com/explore`。

更新代码后，需要回到 `edge://extensions/` 点一次插件卡片上的「重新加载」。这版为了实现真实自动填写，使用了 Edge 的 `debugger` 权限；如果 Edge 弹出权限确认，选择允许。

## 使用

- 右侧面板里输入关键词，每行一个。
- 在「评论草稿池」里添加多个草稿；每个草稿框都可以写多行。
- 点击「扫当前」只扫描已经加载在页面里的笔记。
- 点击「采集更多」会向下滚动，边加载边扫描更多笔记。
- 如果首页刷不到目标内容，点击「打开搜索页」走站内搜索结果，再用「扫当前」或「采集更多」。
- 页面中匹配到的笔记会出现红色描边，右侧列表会汇总命中结果。
- 点击「复制草稿」后，会从草稿池随机复制一个完整草稿，到笔记里自行确认并发布。
- 点击「打开并预填」会随机抽一个完整草稿，打开对应笔记，自动展开评论框，并把草稿写进去；最后仍需要你手动检查并点击小红书原生发布按钮。

## 自动预填说明

- 预填会从「评论草稿池」里随机选 1 条完整草稿。
- 单条草稿可以写多行，插件会按完整草稿写入，不会按每一行随机拆开。
- 插件会尝试自动展开评论框并写入内容，不需要手动按 `Ctrl+V`。
- 插件不会自动点击发布、发送、关注、点赞、收藏等按钮。
- 真实填写依赖 Edge 的 `debugger` 权限；填写完成后插件会主动断开调试连接。

## 扫不到时怎么调

- 先把关键词放宽，比如从完整短句改成 `求推荐`、`怎么买`、`多少钱` 这种词。
- 增大「采集页数」，再点「采集更多」。
- 关键词比较偏时，不要扫首页推荐流，直接用「打开搜索页」走站内搜索结果。
- 如果页面出现验证码或登录拦截，先手动处理页面状态，再继续采集。
